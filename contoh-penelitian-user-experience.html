16. contoh aplikasi yang termasuk User friendly dan Bukan Termasuk User friendly​


Jawaban:

Pengertian User Friendly

User friendly adalah suatu keadaan dimana ketika seseorang menggunakan alat, software, website atau sistem operasi secara mudah. Saat ini, segala sesuatu hal hatus user friendly karena manusia cenderung selalu mencari kemudahan dalam kehidupan mereka. Lalu jika sebuah aplikasi android atau browser user friendly, berarti orang yang menggunakan aplikasi tersebut merasa mudah dan tidak mengalami kesulitan.

Kenyataannya, kenyamanan pengguna aplikasi tidak dapat dianggap sepele. Kemudahan dan menjadikan pengguna aplikasi sebagai rumah saat menjalani aplikasi adalah sebuah keharusan. Sebuah aplikasi tidak boleh terlalu memusingkan serta menyulitkan pengguna karena fitur-fitur atau fungsionalitas yang kurang berguna.

Ciri-Ciri Aplikasi Keuangan Android yang User Friendly

1. Mudah Digunakan

Mudah digunakan adalah salah satu alasan utama mengapa sebuah aplikasi keuangan android bisa disebut user friendly. Kemudahan dalam sebuah aplikasi keuangan bisa dilihat dari berbagai kemudahan bagi pengguna untuk mengakses setiap fitur yang dimilikinya. Jika sebuah aplikasi memiliki fitur yang tidak rumit, namun mampu menghasilkan segala kebutuhan laporan keuangan secara multi-fungsi, maka bisa di katakan aplikasi keuangan tersebut cukup bagus dan mudah digunakan.

2. Mudah Dipahami

Tidak semua pengguna aplikasi keuangan adalah orang-orang yang memahami masalah keuangan atau istilah yang sering digunakan dalam masalah keuangan. Jika sebuah aplikasi keuangan memiliki istilah umum yang bisa dipahami oleh setiap orang dengan mudah, maka aplikasi tersebut bisa digunakan oleh orang awam secara luas dan termasuk dalam kategori user friendly.

3. Praktis

Sebuah aplikasi keuangan yang user friendly bukanlah yang memiliki banyak fitur, tapi juga mampu menghasilkan banyak laporan keuangan melalui satu pencatatan yang diperlukan. Banyak orang menggunakan sebuah aplikasi karena tingkat efektivitas dan kepraktisannya yang tinggi. Aplikasi dapat membantu pengguna menyelesaikan sebuah pekerjaan lebih baik dari sebelumnya dan tidak menimbulkan hambatan bagi pengguna aplikasi.

4. Menarik secara Estetika

Mengikuti tren sambil mengorbankan kemudahan pengguna adalah salah besar. Pengguna akan mengalami hal yang tidak menyenangkan. Setiap update yang dilakukan tidak harus mengorbankan keramahan user selama menggunakan aplikasi keuangan. Desain menu dan hal lainnya menarik tetapi tetap menggunakan prinsip keep it simple. Sehingga pengguna terus nyaman selama melihat aplikasi.

5. User Merasa Nyaman saat Menjalankan Aplikasi

Ketika keempat ciri-ciri diatas sudah terpenuhi, maka dengan sendirinya pengguna akan merasa senang dan nyaman dalam menggunakan aplikasi. Tidak perlu repot sehingga tidak pindah ke lain hati. Kenyamanan user juga didukung dengan perbaikan dan kemajuan yang dilakukan secara berkala oleh aplikasi sehingga user akan terus nyaman dalam menggunakan aplikasi meskipun bertahun-tahun lamanya.

27. 1. Sebutkan ada berapa macam user interface, dan berilah contoh masing-masing user interface tersebut.​


Jawaban:

Ada beberapa macam antarmuka pengguna (user interface/UI), dan berikut adalah beberapa contoh:

1. Antarmuka Pengguna Grafis (GUI):

Contoh: Windows, macOS, dan banyak sistem operasi modern menggunakan GUI yang memungkinkan pengguna berinteraksi dengan elemen visual seperti ikon, tombol, dan jendela.

2. Antarmuka Pengguna Command-Line (CLI):

Contoh: Command Prompt di Windows, Terminal di macOS/Linux. Pengguna berinteraksi dengan sistem melalui baris perintah atau sintaksis teks untuk menjalankan perintah.

3. Antarmuka Pengguna Berbasis Teks:

Contoh: Pengedit teks seperti Notepad atau Vim. Pengguna berinteraksi langsung dengan teks untuk membuat atau mengedit informasi.

4. Antarmuka Pengguna Berbasis Web (Web UI):

Contoh: Situs web dan aplikasi web. Pengguna berinteraksi dengan elemen antarmuka yang ditampilkan di browser web.

5. Antarmuka Pengguna Berbasis Suara (Voice User Interface/VUI):

Contoh: Asisten suara seperti Siri, Google Assistant, atau Alexa. Pengguna berkomunikasi dengan sistem melalui perintah suara.

6. Antarmuka Pengguna Berbasis Sentuhan (Touch User Interface/TUI):

Contoh: Layar sentuh pada perangkat mobile seperti smartphone atau tablet. Pengguna berinteraksi dengan menyentuh layar untuk melakukan berbagai tindakan.

7. Antarmuka Pengguna Berbasis Realitas Virtual (Virtual Reality User Interface/VR UI):

Contoh: Pengalaman virtual reality di headset VR. Pengguna berinteraksi dengan dunia virtual menggunakan gerakan atau kontroler khusus.

8. Antarmuka Pengguna Berbasis Gerak (Gesture User Interface):

Contoh: Microsoft Kinect atau teknologi gestur pada perangkat mobile. Pengguna berinteraksi dengan perangkat melalui gerakan fisik atau gestur tertentu.

Setiap jenis antarmuka pengguna memiliki karakteristik unik dan memenuhi kebutuhan berbagai konteks penggunaan.

29. Lakukan simulasi UX research dengan menggunakan user persona& customer experience map pada apk Spotify


Penjelasan:

Tentu, berikut adalah simulasi UX research dengan menggunakan user persona dan customer experience map pada aplikasi Spotify:

**User Persona**

Nama: Andi

Usia: 25 tahun

Jenis kelamin: Laki-laki

Pekerjaan: Mahasiswa

Hobi: Mendengarkan musik, bermain game, dan menonton film

Andi adalah seorang mahasiswa yang berusia 25 tahun. Ia suka mendengarkan musik, bermain game, dan menonton film. Andi menggunakan Spotify untuk mendengarkan musik. Ia menyukai Spotify karena memiliki banyak pilihan lagu, mudah digunakan, dan gratis. Andi juga menyukai fitur-fitur yang ada di Spotify, seperti Discover Weekly, Top Songs, dan Radio.

**Customer Experience Map**

Customer experience map adalah alat yang digunakan untuk menggambarkan pengalaman pelanggan saat berinteraksi dengan produk atau layanan. Customer experience map dapat membantu perusahaan untuk memahami kebutuhan dan harapan pelanggan, serta untuk meningkatkan pengalaman pelanggan.

Berikut adalah customer experience map untuk aplikasi Spotify:

* **Tujuan:** Andi ingin mendengarkan musik yang ia sukai.

* **Tindakan:** Andi membuka aplikasi Spotify dan mencari lagu yang ia sukai.

* **Emosi:** Andi merasa senang dan puas saat mendengarkan musik yang ia sukai.

* **Hasil:** Andi dapat mendengarkan musik yang ia sukai dengan mudah dan nyaman.

**Kesimpulan**

User persona dan customer experience map adalah alat yang penting untuk melakukan UX research. User persona dapat membantu perusahaan untuk memahami kebutuhan dan harapan pelanggan, sedangkan customer experience map dapat membantu perusahaan untuk meningkatkan pengalaman pelanggan.

itulah yang disebut dengan Percieved value pricing

Penjelasan:

Percieved value pricing adalah dimana pelanggan bersedia membayar suatu produk berdasarkan persepsi mereka tentang produk tersebut. Percieved value pricing tdk berdasar pada cost-of-product melainkan pada nilai2 yang dipercaya oleh pelanggan, akan mereka dapatkan dari produk tersebut. Percieved value pricing termasuk marketing strategy yang penting bagi perusahaan karena membantu perusahaan dalam menentukan harga. Contohnya adalah jam tangan. Walaupun cost-of production mereka rendah. Perusahaan jam tangan tetap menjual jam tangan mereka dengan harga tinggi. Pelanggan pun tetap rela untuk membeli jam tangan tersebut karena "percieved value" yg mereka punya, seperti experience dan status sosial yang bisa mereka dapatkan.

39. Lakukan simulasi UX research dengan menggunakan user persona & customer experience map​


Jawaban:

1. **Buat persona pengguna.** Persona pengguna adalah karakter fiksi yang mewakili pengguna target Anda. Ini harus mencakup informasi tentang demografi pengguna, tujuan, kebutuhan, dan masalah yang dihadapi.

2. **Buat peta pengalaman pelanggan.** Peta pengalaman pelanggan adalah representasi visual perjalanan pelanggan dengan produk atau layanan Anda. Ini harus menunjukkan berbagai titik kontak antara pelanggan dengan produk atau layanan Anda, serta emosi yang mereka alami pada setiap titik kontak.

3. **Gunakan persona pengguna dan peta pengalaman pelanggan sebagai panduan dalam penelitian UX Anda.** Ketika melakukan penelitian UX, gunakan persona pengguna dan peta pengalaman pelanggan untuk fokus pada kebutuhan dan masalah pengguna target Anda. Ini akan membantu Anda mengidentifikasi area di mana Anda dapat meningkatkan pengalaman pengguna dari produk atau layanan Anda.

Berikut adalah contoh simulasi penelitian UX menggunakan persona pengguna dan peta pengalaman pelanggan:

Persona pengguna:

Nama: Jane Doe

Usia: 35 tahun

Pekerjaan: Insinyur perangkat lunak

Tujuan: Menemukan agen perjalanan online yang dapat diandalkan dan terjangkau

Kebutuhan: Agen perjalanan yang menawarkan berbagai pilihan penerbangan dan hotel, serta harga yang kompetitif

Masalah yang dihadapi: Sulit menemukan penerbangan atau hotel yang tepat

Peta pengalaman pelanggan:

Titik Kontak | Emosi

------- | --------

Mengunjungi situs web | Semangat untuk mulai merencanakan perjalanan

Mencari penerbangan | Frustrasi dengan banyaknya hasil pencarian

Membandingkan harga | Bingung dengan berbagai pilihan harga

Memesan penerbangan | Senang telah menemukan penawaran bagus

Check-in untuk penerbangan | Cemas tentang keberangkatan penerbangan

Naik penerbangan | Santai dan antusias menjalani perjalanan

Tiba di tujuan | Senang sedang berlibur

Dalam contoh ini, persona pengguna dan peta pengalaman pelanggan dapat digunakan untuk memandu penelitian UX. Misalnya, peneliti dapat melakukan pengujian kegunaan dengan pengguna untuk melihat seberapa mudah menemukan penerbangan dan hotel di situs web. Peneliti juga dapat melakukan wawancara dengan pengguna untuk mendapatkan umpan balik tentang pilihan harga dan proses pemesanan.

Hasil penelitian UX dapat digunakan untuk meningkatkan pengalaman pengguna situs web. Misalnya, peneliti dapat merekomendasikan agar situs web memudahkan perbandingan harga dan memberikan informasi lebih lanjut tentang proses pemesanan.

Penelitian UX adalah bagian penting dari proses desain UX. Dengan menggunakan persona pengguna dan peta pengalaman pelanggan, peneliti UX dapat fokus pada kebutuhan dan masalah pengguna target mereka. Ini dapat membantu memastikan bahwa produk atau layanan akhir ramah pengguna dan memenuhi kebutuhan penggunanya.
