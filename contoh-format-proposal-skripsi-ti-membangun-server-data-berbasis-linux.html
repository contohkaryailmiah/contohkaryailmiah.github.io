Jawaban: Sistem Operasi Jaringan Berbasis GUI (Grafical User Interface)

Sistem Operasi Jaringan Berbasis GUI (Grafical User Interface) merupakan Sistem operasi komputer yang dalam proses Instalasi dan adminitrasinya, user tidak perlu menghafal sintax – sintax atau perintah DOS atau bahasa pemograman yang digunakannya. karena siste operasi tersebut sudah berbasis grafis sehingga memudahkan dalam penggunaanya. Kebanyakan jenis sistem operasi ini adalah sistem oprasi dari keluarga microsoft yang memang mengedepankan tampilan grafis yang mudah dipahami sebagai adalanya. Berikut beberapa contoh Sistem Operasi jaringan berbasis GUI: Linux Redhat, Windows NT 3.51, Windows 2000 (NT 5.0), Windows Server 2003, Windows XP, Microsoft MS-NET, Microsoft LAN Manager, Novell NetWare.

Sistem Operasi Jaringan Berbasis Text/CLI (Command Line Interface)

Sistem Operasi Jaringan Berbasis Text (Command Line Interface) merupakan sistem operasi yang proses instalasi dan penggunaanya, user diharapkan untuk menghafal perintah DOS atau perintah dasar linux yang digunakan untuk menjalankan suatu proses instalasi. Setiap akan melakukan instalasi paket maka kita akan mengetikan perintah untuk bisa menginstall paket tersebut. Berikut adalah contoh Sistem Operasi Jaringan berbasis text: Linux Debian, Linux Suse, Sun Solaris, Linux Mandrake, Knoppix, MacOS. UNIX, Windows NT, Windows 2000 Server, Windows 2003 Server.

4. apakah semua website berbasis client server? jelaskan dan beri contoh!​


Ya, seluruh website memiliki basis client server. Hal tersebut karena client server memiliki peran sebagai penghubung antara client yaitu pengguna, dengan server sebagai penyedia informasi yang diminta oleh pengguna. Contoh aplikasi yang menggunakan sistem client server adalah email.

Pembahasan

Client server adalah sebuah arsitektur jaringan yang menghubungkan sistem server dan sistem client yang berkomunikasi melalui jaringan komputer. Client adalah perangkat yang menerima dan akan menampilkan serta menjalankan aplikasi. Server adalah perangkat yang menyediakan serta bertindak sebagai pengelola aplikasi, data, dan keamanannya.

Prinsip kerja client server bergerak dua arah dan bertugas sebagai program web browser yang mengirimkan informasi kepada user yang ada di seluruh dunia. Untuk mengoperasikannya, client server membutuhkan IP address dari server dan halaman website.

Penjelasan sistematika proposal skripsi

1. Cover Proposal

Cover proposal merupakan halaman depan yang pertama kali dilihat oleh penguji, cover proposal skripsi berisi judul skripsi, nama penyusun, nomor induk mahasiswa, logo universitas, nama program study, jurusan, nama universitas, dan paling bawah tahun pembuatan.

2. Lembar Judul

Biasanya setiap perguruan tinggi berbeda-beda ada yang pakai ada juga yang tidak pakai, lembar judul ini merupakan lembaran yang hanya berisi judul skripsi yang diajukan.

3. Lembar Pengesahan

Lembar pengesahan merupakan halaman untuk mengesahkan skripsi yang dibuat, biasanya pada uji proposal lembar pengesahan dibuat hanya saja tidak dibubuhi tanda tangan dosen pembimbing. Namun, pengesahan proposal skripsi biasanya hanya berupa acc dari dosen pembimbing utama di halaman cover.

4. Daftar Isi

Daftar isi adalah daftar halaman dari isi proposal skripsi yang dibuat, daftar isi dibuat agar pembaca atau penguji dengan mudah menemukan bagian-bagian dari proposal skripsi.

5. Kata Pengantar

Kata pengantar berisi sebuah pernyataan pengantar dari penulis yang menjelaskan tentang maksud dan tujuan dari penulisan skripsi beserta ucapan terima kasih dari penulis kepada pihak-pihak kampus dan pihak-pihak yang membantu penulis.

6. Daftar Tabel

Daftar tabel merupakan daftar judul-judul tabel yang berada pada proposal skripsi, daftar tabel berfungsi untuk mempermudahkan pembaca dalam mencari tabel yang berkaitan dengan pembahasan.

7. Daftar Gambar

Daftar gambar merupakan daftar-daftar gambar yang berada pada selusuh isi dari proposal, daftar gambar dipakai jika proposal skripsi terdapat gambar jika tidak terdapat gambar sebaiknya jangan sisipkan lembaran daftar gambar.

8. Daftar Lampiran

Daftar lampiran merupakan halaman yang berisi daftar-daftar lampiran seperti lampiran surat penelitian, lampiran berkas, dan lampiran-lampiran yang dibutuhkan dalam penyelesaian skripsi.

9. BAB 1

Bab 1 merupakan bab pendahuluan dari proposal skripsi yang didalamyan berisi latar belakang masalah, rumusan masalah, batasan masalah, tujuan penelitian, serta manfaat penelitian.

10. BAB 2

Bab 2 merupakan bab isi dari proposal, disarankan pada landasan teori jangan terlalu banyak hanya yang dibutuhkan saja untuk proposal.

11. BAB 3

Bab 3 merupakan bab yang berisi metode penelitian seperti lokasi dan waktu penelitian, teknik pengumpulan data, teknik analisis data dan juga analisis sistem.

12. Daftar Pustaka

Daftar pustaka berisi daftar bahan-bahan referensi atau literature yang telah digunakan untuk penelitian. Referensi tersebut baik diambil dari buku, jurnal, maupun sumber elektronik maka harus disertakan dalam halaman daftar pustaka. Baca panduannya di artikel 

13. Lampiran

Lampiran merupakan halaman yang berisi scanan atau potokopi lampiran berkas-berkas yang dibutuhkan untuk melengkapi data-data penelitian dalam tugas akhir. Baik berupa surat penelitian ataupun berkas-berkas dari tempat penelitian.

8. MySQL server lebih ringan dan cepat jika diinstal pada sistem operasi Linux. Layanan web server Apache akan lebih cepat dan ringan pada server berbasis Linux. 1. Buatlah konklusi dari premis tersebut menggunakan penalaran deduktif.2. buatlah analisis kesesuaian penalaran deduktif terhadap premis tersebut.mohon dijawab terimakasih ​


Jawaban:

Konklusi dari Premis (Penalaran Deduktif)

Oleh karena MySQL server lebih ringan dan cepat jika diinstal pada sistem operasi Linux, serta layanan web server Apache akan lebih cepat dan ringan pada server berbasis Linux, maka dapat disimpulkan bahwa menggunakan Linux sebagai sistem operasi untuk hosting MySQL server dan layanan web server Apache akan memberikan keuntungan dalam hal kinerja yang lebih baik dan efisien.

Analisis Kesesuaian Penalaran Deduktif

Penalaran deduktif melibatkan pengambilan kesimpulan yang logis berdasarkan premis atau asumsi yang diberikan. Dalam hal ini, premis yang diberikan adalah bahwa MySQL server lebih ringan dan cepat di Linux serta Apache lebih cepat dan ringan di Linux. Konklusi yang diambil adalah bahwa menggunakan Linux sebagai sistem operasi untuk hosting MySQL dan Apache akan memberikan keuntungan dalam kinerja.

Namun, perlu diingat bahwa penalaran deduktif hanya memberikan kesimpulan yang sah jika premis yang diberikan benar. Analisis kesesuaian tergantung pada kebenaran premis tersebut. Jika premis-premis yang diberikan benar, maka konklusi yang diambil adalah sah. Namun, jika premis-premis tersebut tidak benar atau hanya berlaku dalam situasi tertentu, maka kesimpulan tersebut mungkin tidak valid.

Dalam hal ini, untuk mendukung kesimpulan tersebut, perlu ada bukti yang kuat yang menunjukkan bahwa MySQL dan Apache memang lebih ringan dan cepat di Linux. Selain itu, asumsi bahwa faktor-faktor lain tidak mempengaruhi kinerja (seperti konfigurasi server dan sumber daya yang tersedia) juga harus dipertimbangkan. Jika premis-premis tersebut dapat ditegakkan dan tidak ada faktor lain yang signifikan, maka penalaran deduktif tersebut dapat dianggap sesuai.

Perangkat lunak yang digunakan untuk mengelola dan memanggil kueri (query) basis data disebut sistem manajemen basis data (database management system, DBMS). Sistem basis data dipelajari dalam ilmu informasi.

Istilah "basis data" berawal dari ilmu komputer. Meskipun kemudian artinya semakin luas, memasukkan hal-hal di luar bidang elektronika, artikel ini mengenai basis data komputer. Catatan yang mirip dengan basis data sebenarnya sudah ada sebelum revolusi industri yaitu dalam bentuk buku besar, kuitansi dan kumpulan data yang berhubungan dengan bisnis.

Konsep dasar dari basis data adalah kumpulan dari catatan-catatan, atau potongan dari pengetahuan. Sebuah basis data memiliki penjelasan terstruktur dari jenis fakta yang tersimpan di dalamnya: penjelasan ini disebut skema. Skema menggambarkan objek yang diwakili suatu basis data, dan hubungan di antara objek tersebut. Ada banyak cara untuk mengorganisasi skema, atau memodelkan struktur basis data: ini dikenal sebagai model basis data atau model data. Model yang umum digunakan sekarang adalah model relasional, yang menurut istilah layman mewakili semua informasi dalam bentuk tabel-tabel yang saling berhubungan di mana setiap tabel terdiri dari baris dan kolom (definisi yang sebenarnya menggunakan terminologi matematika). Dalam model ini, hubungan antar tabel diwakili denga menggunakan nilai yang sama antar tabel. Model yang lain seperti model hierarkis dan model jaringan menggunakan cara yang lebih eksplisit untuk mewakili hubungan antar tabel.

Penjelasan:

Lingkungan basis data adalah sebuah habitat di mana terdapat basis data untuk bisnis. Dalam lingkungan basis data, pengguna memiliki alat untuk mengakses data. Pengguna melakukan semua tipe pekerjaan dan keperluan mereka bervariasi seperti menggali data (data mining), memodifikasi data, atau berusaha membuat data baru. Masih dalam lingkungan basis data, pengguna tertentu tidak diperbolehkan mengakses data, baik secara fisik maupun logis. (Koh, 2005, dalam Janner Simarmata & Imam Paryudi 2006: 33).

31. Server yang dapat membuat sistem berbasis unix (seperti linux) untuk melakukan sharing resource dengan sistem berbasi windows adalah...


Server yang bisa membuat platform dengan sistem operasi Linux/Unix bisa berbagi resource dengan perangkat yang menggunakan Windows adalah Samba Server.

Pembahasan:

Samba Server bertujuan untuk menjadi jembatan bagi platform dengan sistem operasi Linux dan Windows bisa berkomunikasi dan dijalankan dalam satu jaringan komputer.

Karena itu, software untuk Samba Server sendiri bisa berjalan baik di OS Linux ataupun Windows yang menggunakan SMB (Server Message Block). SMB sendiri adalah protokol komunikasi data yang banyak dipakai oleh Microsoft dan Linux untuk membantu proses sharing file, resource, dan printer serta tugas-tugas lain yang berhubungan dengan jaringan.

Fungsi samba server adalah sebagai berikut:

Menghubungkan sistem dengan OS Linux dan mesin Windows: Kedua OS yang berbeda ini membutuhkan perantara agar dapat terhubung dalam suatu jaringan. Baik untuk berkirim data, berbagi sumber daya device, menggunakan firewall, DNS, DHCP, FTP, dan lain-lain. Melakukan verifikasi user untuk setiap klien yang bergabung dengan domain jaringan tertentu. Samba memiliki fitur Samba PDC (Primary Domain Controller) yang bisa melakukan hal ini. Menjadi domain controller pada perangkat jaringan dengan OS Windows.

Keunggulan samba server adalah sebagai berikut:

1. Dapat digunakan secara gratis (open-source).

2. Bisa berjalan secara multi-platform.

3. Mengkonfigurasikannya sangat mudah.

4. Jarang mengalami error dalam penggunaan untuk jaringan.

5. Memiliki banyak fitur untuk mengatur komputer-komputer dalam suatu jaringan.
